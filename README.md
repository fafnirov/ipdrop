# IpDrop

Открытый аналог AirDrop для Windows: отправляйте фото и файлы с iPhone (или любого
устройства) на компьютер по Wi-Fi — без проводов, без облака, без аккаунтов.

Это **не** настоящий AirDrop (его протокол AWDL закрыт Apple и на Windows недоступен),
а простой приёмник в духе LocalSend, который реально работает на Windows: ПК поднимает
локальную веб-страницу, телефон отправляет на неё файлы.

> **Почему не нативное iOS-приложение?** Сборка и установка приложения на iPhone
> требуют Mac + Xcode и подписи кода (Apple Developer). Вместо этого IpDrop
> использует значок «На экран Домой» + постоянный адрес `ipdrop.local` — получается
> почти как приложение, но без Mac и платного аккаунта.

## Возможности

- 🖥️ Окно-панель (на базе встроенного в Windows WebView2): статус, QR-код, живой
  список принятых файлов с превью картинок, настройки — без чёрной консоли
- 📥 Приём фото и файлов с iPhone/Android/любого устройства по Wi-Fi
- 📲 Значок на экране «Домой» — открывается одним тапом, сканировать каждый раз не нужно
- 🏷️ Постоянный адрес `ipdrop.local` (через mDNS/Bonjour) — не зависит от смены IP
- 🗂️ Несколько файлов за раз, большие видео (потоковая запись), без перезаписи одноимённых
- 🚀 Автозапуск при входе в Windows (в фоне, без окна)
- 🔒 Команды «открыть папку» и «автозапуск» доступны только с самого ПК, не с телефона
- 📦 Один `.exe` без внешних зависимостей рантайма

## Быстрый старт

1. Запустите `ipdrop.exe` на ПК — откроется окно IpDrop с QR-кодом.
2. При первом запуске Windows спросит про брандмауэр — **разрешите «Частные сети»**.
3. На iPhone откройте «Камеру» → наведите на QR-код в окне → откройте ссылку.
   (ПК и телефон должны быть в одной Wi-Fi-сети.)
4. На странице нажмите **«Поделиться» → «На экран Домой»** — появится значок IpDrop.
5. Дальше: тап по значку → выбрать фото/файлы → они появятся в окне и в папке на ПК.

## Папка приёма

По умолчанию: `%USERPROFILE%\IpDrop` (например `C:\Users\Имя\IpDrop`).
Изменить: `ipdrop.exe -dir "D:\Принятое"`

## Автозапуск

```
ipdrop.exe -install-autostart      :: включить (с текущими -port и -dir)
ipdrop.exe -uninstall-autostart    :: выключить
```

## Флаги

| Флаг                   | Назначение                            | По умолчанию             |
|------------------------|---------------------------------------|--------------------------|
| `-port`                | TCP-порт веб-сервера                  | `8080`                   |
| `-dir`                 | папка для сохранения файлов           | `%USERPROFILE%\IpDrop`   |
| `-headless`            | фоновый режим без окна (для автозапуска) | —                     |
| `-install-autostart`   | добавить в автозапуск и выйти         | —                        |
| `-uninstall-autostart` | убрать из автозапуска и выйти         | —                        |

## Если телефон не открывает страницу

- Проверьте, что ПК и iPhone в **одной** Wi-Fi-сети (не гостевой/изолированной).
- Если `ipdrop.local` не открывается — используйте запасной адрес по IP (показан в окне).
- Разрешите порт в брандмауэре вручную (PowerShell от администратора):
  ```powershell
  New-NetFirewallRule -DisplayName "IpDrop" -Direction Inbound `
    -Protocol TCP -LocalPort 8080 -Action Allow -Profile Private
  ```

## Сборка из исходников

Нужен [Go](https://go.dev/dl/) 1.26+. Сборка как GUI-приложение (без окна консоли):

```
go build -ldflags "-H windowsgui" -o ipdrop.exe .
```

Или просто запустите `build.bat`.

## Как это устроено

| Файл                   | Назначение                                        |
|------------------------|---------------------------------------------------|
| `main.go`              | запуск, режимы (окно / `-headless`), single-instance |
| `config.go`            | флаги командной строки                            |
| `web.go`               | HTTP-сервер, приём файлов, приватный API           |
| `page.go`              | мобильная страница приёма (для телефона)           |
| `panel.go`             | панель управления в окне на ПК                     |
| `gui_windows.go`       | окно приложения через WebView2                     |
| `console_windows.go`   | вывод в терминал для GUI-сборки                    |
| `icon.go`              | генерация иконки приложения и манифест PWA         |
| `mdns.go`              | публикация имени `ipdrop.local`                    |
| `autostart_windows.go` | автозапуск через реестр Windows                    |
